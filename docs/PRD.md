# Product Requirements Document (PRD)
## LinkLayer - Умный блокнот с системой графов

**Версия:** 1.0  
**Дата создания:** 2026-03-14  
**Статус:** В разработке (MVP)

---

## 1. Обзор продукта

**LinkLayer** — это веб-приложение для управления знаниями и идеями, вдохновленное Obsidian, но с более интуитивным интерфейсом. Приложение позволяет пользователям создавать, организовывать и визуализировать связи между заметками через интерактивный граф.

### Основная ценность
- **Простота использования**: Интуитивный UI, понятнее чем Obsidian
- **Визуализация знаний**: Два режима графов для разных стилей работы
- **Совместная работа**: Возможность создавать общие хранилища для команд
- **Гибкость**: Ручное соединение заметок, группировка, поиск по тегам

---

## 2. Целевая аудитория

- **Первичная**: Все желающие (студенты, исследователи, писатели, профессионалы)
- **Вторичная**: Небольшие компании и инди-разработчики (для совместной работы)

### Проблемы, которые решает LinkLayer
- Сложность организации больших объемов информации
- Необходимость визуализации связей между идеями
- Отсутствие простого инструмента для совместной работы над знаниями
- Сложность экспорта и импорта данных

---

## 3. Ключевые возможности MVP

### 3.1 Управление заметками
- ✅ Создание, редактирование, удаление заметок
- ✅ Формат хранения: Markdown (.md)
- ✅ Возможность импорта .md файлов
- ✅ Возможность экспорта заметок в .md
- ✅ Система тегов для категоризации

### 3.2 Система хранилищ (Vaults)
- ✅ Личное хранилище для каждого пользователя
- ✅ Доступ через аутентификацию (Google OAuth и другие сервисы)
- ✅ Возможность создания совместных хранилищ
- ✅ Управление доступом (создатель может добавлять соавторов)

### 3.3 Граф и визуализация
- ✅ **Режим 1 (Классический)**: Прямые линии между узлами (как в Obsidian)
- ✅ **Режим 2 (Физический)**: Интерактивный граф с физикой (узлы отталкиваются, связи прогибаются)
- ✅ Возможность закрепить заметку на месте
- ✅ Ручное соединение узлов
- ✅ Плавная анимация перемещения
- ✅ Поиск по тегам и фильтрация по группам
- ✅ Полупрозрачность отфильтрованных заметок для акцента

### 3.4 Поиск и навигация
- ✅ Поиск по тегам
- ✅ Фильтрация по группам
- ✅ Быстрая навигация между связанными заметками

### 3.5 Совместная работа (базовая версия)
- ✅ Добавление соавторов в хранилище
- ✅ Равноправный доступ соавторов (могут добавлять заметки)
- ✅ Встроенный чат в хранилище
- ✅ Возможность прикрепить заметку к сообщению в чате

## 4. Требования к функциональности

### 4.1 Аутентификация и авторизация
- Регистрация через Google OAuth
- Регистрация через другие OAuth провайдеры (GitHub, Microsoft и т.д.)
- Сохранение профиля пользователя (username, способ регистрации)
- Управление сессиями

### 4.2 Редактор заметок
- Редактор Markdown с предпросмотром
- Поддержка базового форматирования (заголовки, списки, код, ссылки)
- Автосохранение
- Возможность добавления ссылок на другие заметки

### 4.3 Система графов
- **Режим 1**: Статический граф с прямыми линиями
- **Режим 2**: Динамический граф с физическим движком
  - Узлы отталкиваются друг от друга
  - Связи прогибаются под "весом"
  - Возможность "закрепить" узел кнопкой
  - Зум и панорамирование
  
### 4.4 Управление связями
- Создание связей между заметками вручную
- Удаление связей
- Типы связей (опционально для MVP)
- Визуализация связей в графе

### 4.5 Теги и группы
- Создание и управление тегами
- Присвоение тегов заметкам
- Фильтрация по тегам в графе
- Группировка заметок

### 4.6 Совместная работа
- Создание совместного хранилища
- Приглашение соавторов (по email или ссылке)
- Синхронизация изменений в реальном времени
- Встроенный чат с поддержкой упоминания заметок
- Просмотр активности в хранилище

### 4.7 Импорт/Экспорт
- Импорт .md файлов в хранилище
- Экспорт отдельных заметок в .md
- Экспорт всего хранилища в архив

## 5. Архитектура данных

### 5.1 Основные сущности

#### User
```
- id (UUID)
- username (string)
- email (string)
- auth_provider (string: google, github, microsoft)
- created_at (timestamp)
- updated_at (timestamp)
```

#### Vault (Хранилище)
```
- id (UUID)
- name (string)
- owner_id (UUID, FK to User)
- is_shared (boolean)
- created_at (timestamp)
- updated_at (timestamp)
```

#### Note (Заметка)
```
- id (UUID)
- vault_id (UUID, FK to Vault)
- title (string)
- content (text, markdown)
- created_at (timestamp)
- updated_at (timestamp)
- created_by (UUID, FK to User)
```

#### Tag (Тег)
```
- id (UUID)
- vault_id (UUID, FK to Vault)
- name (string)
- color (string, optional)
```

#### NoteTag (Связь заметка-тег)
```
- note_id (UUID, FK to Note)
- tag_id (UUID, FK to Tag)
```

#### Link (Связь между заметками)
```
- id (UUID)
- source_note_id (UUID, FK to Note)
- target_note_id (UUID, FK to Note)
- link_type (string, optional: "references", "related", "depends_on")
- created_at (timestamp)
```

#### VaultMember (Участник хранилища)
```
- id (UUID)
- vault_id (UUID, FK to Vault)
- user_id (UUID, FK to User)
- role (string: "owner", "editor")
- joined_at (timestamp)
```

#### ChatMessage (Сообщение в чате)
```
- id (UUID)
- vault_id (UUID, FK to Vault)
- user_id (UUID, FK to User)
- content (text)
- attached_note_id (UUID, FK to Note, optional)
- created_at (timestamp)
```

## 6. Технический стек

### Frontend
- **Framework**: React 18+
- **Meta-framework**: Next.js 14+
- **Язык**: TypeScript
- **Стилизация**: TailwindCSS / CSS Modules
- **Граф**: D3.js или Cytoscape.js (для режима 1), Babylon.js или Three.js (для режима 2 с физикой)
- **Состояние**: Redux Toolkit / Zustand
- **HTTP клиент**: Axios / Fetch API

### Backend
- **Runtime**: Node.js
- **Framework**: Next.js API Routes / Express.js
- **БД**: PostgreSQL (через Supabase)
- **Аутентификация**: Supabase Auth (OAuth)
- **Real-time**: Supabase Realtime / WebSocket
- **Хранилище файлов**: Supabase Storage

### DevOps
- **Пакетный менеджер**: Bun
- **Линтинг**: Biome
- **Версионирование**: Git

## 10. Метрики и аналитика

### Для MVP собираем минимум:
- Username пользователя
- Способ регистрации (OAuth провайдер)
- Дата создания аккаунта

### Не собираем:
- Поведенческие данные
- Содержимое заметок
- IP адреса

---

## 11. Безопасность

- Все данные передаются по HTTPS
- Аутентификация через OAuth (безопасные токены)
- Доступ к хранилищам только для авторизованных пользователей
- Валидация входных данных на сервере
- CORS настройки для защиты от несанкционированного доступа
